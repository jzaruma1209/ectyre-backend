# BookApp — Google OAuth 2.0 Workflow de Implementación

> Documento para agente de implementación. Guía paso a paso para agregar Login con Google al proyecto BookApp (Node.js + Express + Sequelize + PostgreSQL).

---

## Estado actual del proyecto

| Qué existe | Estado |
|---|---|
| Backend Node.js + Express con JWT | ✅ Funcionando |
| PostgreSQL + Sequelize ORM | ✅ Funcionando |
| Sistema de roles (admin, moderator, guest, user) | ✅ Funcionando |
| Autenticación propia email/password | ✅ Funcionando |
| Login con Google OAuth 2.0 | ❌ No implementado |

**Lo que se va a agregar:** login con Google como método alternativo. Los usuarios que entren con Google se guardan en la misma tabla `Users`. Al autenticarse se genera un JWT idéntico al flujo actual. El resto de la app (reservas, reviews, etc.) no cambia.

---

## Arquitectura de la solución

```
[Usuario]
    │
    ▼
[Google Cloud] ── OAuth 2.0 + OpenID Connect ── CLIENT_ID + CLIENT_SECRET
    │
    ▼
[Passport.js] ── passport-google-oauth20 ── serialize/deserialize
    │
    ▼
[Tu servidor Node.js]
    ├── src/app.js          → agregar session + passport
    ├── src/routes/index.js → agregar ruta /auth
    ├── src/routes/auth.router.js  → NUEVO
    ├── src/config/passport.js     → NUEVO
    └── src/models/user.js  → agregar campo googleId
    │
    ▼
[PostgreSQL] ── tabla Users + columna googleId (nueva migración)
```

## FASE 1 — Tareas manuales del desarrollador

> ⚠️ El agente NO puede hacer estas tareas. Requieren acceso a Google y credenciales reales. El desarrollador debe completarlas ANTES de que el agente empiece.

### 1.1 Crear proyecto en Google Cloud Console

1. Ir a https://console.cloud.google.com
2. Crear un proyecto nuevo o usar uno existente
3. Ir al menú: **APIs & Services → Credentials**
4. Clic en **"Create Credentials" → "OAuth 2.0 Client ID"**
5. Tipo de aplicación: **Web application**
6. En **Authorized redirect URIs** agregar exactamente:
   ```
   http://localhost:8080/api/v1/auth/google/callback
   ```
   > Para producción también agregar la URL de Vercel con el mismo path.
7. Guardar y copiar el `CLIENT_ID` y el `CLIENT_SECRET`

### 1.2 Agregar variables al archivo .env

Agregar estas líneas al `.env` existente del proyecto:

```env
GOOGLE_CLIENT_ID=tu_client_id_de_google_aqui
GOOGLE_CLIENT_SECRET=tu_client_secret_de_google_aqui
SESSION_SECRET=una_cadena_aleatoria_larga_y_segura
CLIENT_URL=http://localhost:3000
```

> `SESSION_SECRET` puede ser cualquier string largo y aleatorio, por ejemplo: `bookapp_super_secret_key_2024_xkz9p`

### Tarea 2.2 — Crear `src/config/passport.js`

Crear el archivo `src/config/passport.js` con este contenido:

```js
const passport = require('passport');
const GoogleStrategy = require('passport-google-oauth20').Strategy;
const { User } = require('../models');
const jwt = require('jsonwebtoken');

passport.use(new GoogleStrategy({
  clientID: process.env.GOOGLE_CLIENT_ID,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET,
  callbackURL: '/api/v1/auth/google/callback'
}, async (accessToken, refreshToken, profile, done) => {
  try {
    const [user] = await User.findOrCreate({
      where: { googleId: profile.id },
      defaults: {
        firstName: profile.name.givenName,
        lastName: profile.name.familyName,
        email: profile.emails[0].value,
        role: 'user',
      }
    });
    return done(null, user);
  } catch (err) {
    return done(err, null);
  }
}));

passport.serializeUser((user, done) => done(null, user.id));
passport.deserializeUser(async (id, done) => {
  const user = await User.findByPk(id);
  done(null, user);
});
```

> ⚠️ Los campos `firstName`, `lastName` deben coincidir con los nombres reales en el modelo `User`. Si el modelo usa otros nombres (como `name`, `username`), ajustar en consecuencia.

**Verificación:**

### Tarea 2.3 — Crear `src/routes/auth.router.js`

Crear el archivo `src/routes/auth.router.js`:

```js
const express = require('express');
const passport = require('passport');
const jwt = require('jsonwebtoken');
const router = express.Router();

// Paso 1: redirige al usuario a Google
router.get('/google',
  passport.authenticate('google', { scope: ['profile', 'email'] })
);

// Paso 2: Google redirige de vuelta aquí
router.get('/google/callback',
  passport.authenticate('google', { failureRedirect: '/api/v1/auth/failure' }),
  (req, res) => {
    // Generar JWT igual que el flujo normal de login
    const token = jwt.sign(
      { id: req.user.id, role: req.user.role },
      process.env.TOKEN_SECRET,
      { expiresIn: process.env.TOKEN_EXPIRES_IN || '1d' }
    );
    // Redirigir al frontend con el token
    const clientUrl = process.env.CLIENT_URL || 'http://localhost:3000';
    res.redirect(`${clientUrl}/auth/callback?token=${token}`);
  }
);

router.get('/failure', (req, res) => {
  res.status(401).json({ error: 'Autenticacion con Google fallida' });
});

module.exports = router;
```

**Verificación:**
- [ ] Archivo existe en `src/routes/auth.router.js`
- [ ] No hay errores de sintaxis

### Tarea 2.5 — Modificar `src/app.js`

El archivo actual tiene esta estructura. Agregar las líneas marcadas como `// NUEVO`:

```js
const express = require('express');
const helmet = require('helmet');
const cors = require('cors');
const session = require('express-session');   // NUEVO
const passport = require('passport');          // NUEVO
require('dotenv').config();
require('./config/passport');                  // NUEVO — carga la estrategia
const router = require('./routes');
const errorHandler = require('./utils/errorHandler');

const app = express();

app.use(express.json());
app.use(helmet());
app.use(cors());

// Session — requerida por Passport                // NUEVO
app.use(session({                                   // NUEVO
  secret: process.env.SESSION_SECRET,               // NUEVO
  resave: false,                                    // NUEVO
  saveUninitialized: false,                         // NUEVO
}));                                                // NUEVO

app.use(passport.initialize());                     // NUEVO
app.use(passport.session());                        // NUEVO

app.use("/api/v1", router);

app.get("/", (req, res) => {
  return res.send("Welcome to express!")
});

app.use(errorHandler);

module.exports = app;
```

> ⚠️ **Orden crítico:** `express-session` DEBE ir antes de `passport.initialize()`. Passport DEBE ir antes del router `/api/v1`.

### Tarea 2.6 — Crear y ejecutar migración para `googleId`

Generar la migración:

```bash
npx sequelize migration:generate --name add-googleId-to-users
```

Abrir el archivo generado en `src/migrations/` y reemplazar su contenido con:

```js
'use strict';
module.exports = {
  async up(queryInterface, Sequelize) {
    await queryInterface.addColumn('Users', 'googleId', {
      type: Sequelize.STRING,
      allowNull: true,
      unique: true,
    });
  },
  async down(queryInterface) {
    await queryInterface.removeColumn('Users', 'googleId');
  }
};
```

> ⚠️ El nombre `'Users'` debe coincidir con el nombre real de la tabla en la BD. Verificar revisando las migraciones existentes en `src/migrations/`.

Ejecutar la migración:

```bash
npx sequelize db:migrate
```

**Verificación:**
- [ ] La migración corre sin errores
- [ ] `npx sequelize db:migrate --status` muestra `add-googleId-to-users` como `up`

## Problemas comunes y soluciones

| Error | Causa | Solución |
|---|---|---|
| `redirect_uri_mismatch` | La URL del callback en Google Console no coincide | Agregar exactamente `/api/v1/auth/google/callback` en Google Console |
| `Cannot find module 'passport'` | No se instalaron las deps | `npm install passport passport-google-oauth20 express-session` |
| `Error: Failed to serialize user` | Passport no encuentra el id del usuario | Verificar que `User.findOrCreate` retorna el objeto con `.id` |
| `token is not defined` en callback | `TOKEN_SECRET` no está en `.env` | Verificar que `TOKEN_SECRET` está en `.env` (ya existía en el proyecto) |
| `SequelizeUniqueConstraintError` en googleId | Usuario intenta loguearse dos veces | Usar `findOrCreate` en lugar de `create` — ya está en el código |
| `password cannot be null` | El modelo requiere password y Google no lo provee | Hacer `password allowNull: true` en `user.js` |
